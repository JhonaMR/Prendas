Se ha implementado un nuevo tipo de usuario especial llamado "Soporte" con las
siguientes características:

- Rol: soporte (equivalente a admin en permisos)
- Login Code: SOP (3 letras)
- PIN: 3438 (4 dígitos)
- Nombre: Soporte
- Permisos: Acceso completo (igual a admin)
- Protecciones: No se puede editar ni eliminar desde maestras

PASOS PARA IMPLEMENTAR
================================================================================

1. INICIALIZAR USUARIO SOPORTE
   
   Ejecuta el script Node.js:
   
   $ cd Prendas/backend
   $ node scripts/init-soporte-user.js
   
   Salida esperada:
   ✅ Usuario Soporte creado exitosamente:
      ID: [uuid-generado]
      Nombre: Soporte
      Login Code: SOP
      PIN: 3438
      Rol: soporte
      Activo: true

2. VERIFICAR EN BASE DE DATOS
   
   $ psql -U usuario -d inventario
   
   SELECT id, name, login_code, role, active 
   FROM users 
   WHERE login_code = 'SOP';
   
   Resultado esperado: 1 fila con los datos del usuario Soporte

3. PROBAR LOGIN
   
   - Abre la aplicación
   - Ingresa:
     * Login Code: SOP
     * PIN: 3438
   - Deberías tener acceso completo

4. VERIFICAR PROTECCIONES
   
   - Ve a Maestras → Usuarios
   - Busca el usuario "Soporte"
   - Verifica que:
     * Tiene etiqueta "Sistema"
     * Botón "Editar" está deshabilitado
     * Botón "Eliminar" no aparece
     * Fondo es amarillo claro

================================================================================
TROUBLESHOOTING
================================================================================

PROBLEMA: El usuario Soporte no aparece después de ejecutar el script
SOLUCIÓN:
  1. Verifica que el usuario fue creado:
     psql -U usuario -d inventario -c "SELECT * FROM users WHERE login_code = 'SOP';"
  2. Si no aparece, ejecuta el script nuevamente:
     node scripts/init-soporte-user.js

PROBLEMA: No puedo hacer login con Soporte
SOLUCIÓN:
  1. Verifica que el PIN es exactamente: 3438
  2. Verifica que el login code es: SOP (mayúsculas)
  3. Verifica que el usuario está activo en la BD

PROBLEMA: El usuario Soporte no tiene permisos de admin
SOLUCIÓN:
  1. Verifica que el rol es exactamente: soporte (minúsculas)
  2. Recarga la página o limpia el caché
  3. Verifica que el token JWT contiene el rol correcto
